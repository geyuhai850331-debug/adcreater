## 设计哲学

**不做什么：**
- 不用紫色/蓝色作为主强调色（AI 工具最泛滥的颜色）
- 不用纯黑 #000000（Cold gray-black 会让眼睛疲劳）
- 不对所有卡片用同样尺寸（AI 生成的网格感）
- 不在按钮上做渐变或发光（玻璃拟态是 2024 年的 AI 味）
- 不出现 confetti、粒子动画、交错入场（廉价 delight）

**做什么：**
- 暖灰暗色调，像一本好杂志在暗光下翻开
- 琥珀色/暖金作为唯一强调色，克制使用
- 编辑感排版：大图更大，小图更小，留白不平均
- 功能即美感：按钮就是按钮，不需要发光证明自己
- 细节让人发现，不是扑面而来

**参考对象：**
- Linear.app 的暗色模式（暖灰底，克制到几乎无聊但极度好用）
- Arc 浏览器的工具感（不抢内容，但在细节上讲究）
- 日本杂志排版（不对称网格、大字配小字、照片比 UI 更突出）
- 早期 Sketch 和 Figma（工具的自信——不需要花哨来证明价值）

## 色彩体系 v2

```
表面层（从浅到深）:
  --surface-0:      #161616   页面基底（不是纯黑）
  --surface-1:      #1e1e1e   卡片、面板
  --surface-2:      #262626   hover 态、悬浮面板
  --surface-3:      #2e2e2e   按下态、选中背景

文字层:
  --text-primary:   #ececec   正文、标题
  --text-secondary: #9c9c9c   说明文字
  --text-tertiary:  #6b6b6b   辅助信息、placeholder

强调色（琥珀/暖金系）:
  --accent:         #d4a853   主按钮、选中态、链接
  --accent-hover:   #e0bb6a   hover 亮
  --accent-muted:   #3d311c   极弱强调背景（几乎看不出来）

功能色:
  --green:          #5c9c6b   成功（偏灰的绿，不刺眼）
  --red:            #c25450   错误（偏灰的红，不刺眼）
  --yellow:         #c9a84c   警告
  --blue:           #5c88a8   信息

边框:
  --border:         #2a2a2a   几乎和表面融为一体
  --border-visible: #383838   hover 时才看清的边框
```

**为什么不用紫色：**
2024-2025 年几乎每一款 AI 工具都用紫色（或电蓝色）。Cursor、Claude、Perplexity、Midjourney、Gamma——它们审美趋同。一家面向电商卖家的工具不需要挤进"AI 产品俱乐部"的颜色体系。琥珀色让人联想到工艺、材料、实物——比紫色更接近"做电商"这件事。

**为什么不用纯黑：**
OLED 纯黑在 UI 中是种偷懒——它让所有东西对比都拉满，但长时间使用的疲劳感也拉满。暖灰色底让白色文字不用那么亮，整体阅读体验更舒适。

## 字体 v2

```
中文: "阿里巴巴普惠体" (Regular + Medium + Bold 三级即可)
英文: "Inter" (Regular 400, Medium 500, Semibold 600)

不使用西文 + 中文混排时的负 letter-spacing ——
那是 Runway 的品牌特征，不是我们的。
自然字距，让文字呼吸。

字号层级:
  页面标题    28px / weight:600 / line-height:1.2
  区块标题    18px / weight:500 / line-height:1.3
  正文        15px / weight:400 / line-height:1.6
  小字/说明   13px / weight:400 / line-height:1.4
  caption    12px / weight:400 / color:text-tertiary

按钮文字:    14px / weight:500
输入框文字:  15px / weight:400
代码/数据:   13px / "JetBrains Mono", monospace
```

## 间距原则

不使用严格的 4px 倍率系统——那太像框架生成的。
使用 3 个间距档位：

```
tight:   8px   组件内部间距
base:    16px  相关元素间距  
loose:   32px  区块之间

特殊情况用 12px、20px、24px——
不对齐的间距比强撑的 4px 系统更自然。
```

## 组件 v2

```
Button Primary:
  bg: accent (#d4a853)
  color: #161616 (深色文字——琥珀色底上白字反而不清晰)
  radius: 6px
  padding: 8px 18px
  font: 14px/500
  hover: accent-hover
  无阴影，无渐变

Button Secondary:
  bg: transparent
  border: 1px solid border-visible
  color: text-primary
  hover: bg -> surface-2

Button Ghost:
  bg: transparent
  color: text-secondary
  hover: color -> text-primary, bg -> surface-1

Card:
  bg: surface-1
  border: 1px solid border (大部分时候看不见)
  radius: 8px
  hover: border-color -> border-visible
  不同卡片可以有不同的 padding（16px、20px、24px）
  不一定每张卡片都相同高度——让内容决定

Input:
  bg: surface-1
  border: 1px solid border
  radius: 6px
  padding: 10px 14px
  color: text-primary
  placeholder: text-tertiary
  focus: border-color -> accent (唯一用 accent 的地方之一)

Modal:
  bg: surface-1
  border: 1px solid border-visible
  radius: 12px
  backdrop: rgba(0,0,0,0.5)  // 不用 blur——太像 iOS
```

## 全局框架

### 登录/注册后的主工作台

```
┌──────────────────────────────────────────────────────────────┐
│  top bar: 56px, surface-1 半透明 + backdrop-blur             │
│  [Wery]        文案制作 · 文生图 · 图生视频 · 效果预览      💎1,280 │
│                 我的资源 · 发布             [头像]            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                                                              │
│                    主 内 容 区                                │
│                                                              │
│              （不同工具的 UI 在此切换）                        │
│                                                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**顶部导航设计决策：**
- 不用侧边栏。原因：侧边栏是 IDE 的交互模式，不适合创造性工具。
  电商卖家的工作流是线性的（写文案→做图→做视频→预览→发布），
  水平导航更符合这个心理模型。
- 导航项之间没有分隔线，靠间距区分。
- 当前位置用 accent 色下划线（2px），不用背景色块。
- 点数余额始终可见，点击可快速充值。
- 头像用姓名首字，不加载 Gravatar（减少第三方请求）。

### 页面过渡

工具切换时不使用动画。瞬间切换。动画越少，工具感越强。

## 模块一：注册与登录

### 登录页

不采用"左视频右表单"的 Runway 经典布局——那是最被滥用的 AI 工具模式。
改用**居中单卡 + 大面积暖灰留白**。

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│                                                              │
│                                                              │
│                    Wery AI                                    │
│                    为电商卖家准备的 AI 创作工具                 │
│                                                              │
│           ┌─────────────────────────────┐                    │
│           │  手机号 / 邮箱               │                    │
│           │  ─────────────────────────  │                    │
│           │  密码                  👁    │                    │
│           │  ─────────────────────────  │                    │
│           │                             │                    │
│           │  [      登录      ]         │                    │
│           │                             │                    │
│           │  短信验证码登录               │                    │
│           └─────────────────────────────┘                    │
│                                                              │
│          没有账号？注册  →                                    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**关键决策：**
- 背景：surface-0 纯色，不放任何视频/图片。视频 demo 应该是进入产品后才看到的，登录页堆满视觉元素是不自信的表现。
- 卡片宽度 380px，居中。border: 1px solid border，很微弱。
- 品牌名用大字号（32px/600），下方 tagline 用小字 caption 灰色。
- 表单元素间距 tight（8px），标签和输入框一体（label 在输入框内，focus 时上移变小字——或者直接用 placeholder 即可，电商用户不陌生）。
- 登录按钮用 accent 色，文字深色。
- 错误提示不要用红色边框闪烁，改成输入框下方出现一行红色小字 + 输入框边框变红。
- 连续错误 3 次出现滑块验证，用系统原生风格。

### 注册页

和登录页同一张卡片，点"注册"时卡片内容平滑替换（高度自动过渡）。不需要跳转到新页面。

注册字段：手机号 → 获取验证码 → 设置密码 → （可选）邀请码。字段依次出现而非一次性全部展示——减少初始压力。

## 模块二：AI 创作工作台

### 顶部工具栏

```
┌──────────────────────────────────────────────────────────────┐
│  文案制作 · 文生图 · 图生视频 · 效果预览   │ 我的资源 · 发布  │  💎 1,280 │
└──────────────────────────────────────────────────────────────┘
```

- 工具切换无动画，瞬间替换下方内容区。
- 当前工具名下方 2px accent 色下划线。
- 资源库和发布在右侧，用竖线分隔（视觉优先级低于创作工具）。
- 点数余额用数字 + 小菱形图标，hover 显示"充值"入口。

### 工具内顶栏（上下文选项）

每个工具的内容区顶部有一行 48px 的工具栏，放该工具的上下文选项：

```

所有下拉用相同的幽灵按钮样式（bg: transparent, border: 1px solid border），hover 边框变 visible。

```

**设计决策：**
- 左侧输入面板取消。改为顶部通栏输入区——因为文案生成的输入是一段文字，不需要占半屏。让它像一页文档一样自然。
- 输入框高度自适应内容，最少 3 行。聚焦时底部出现字数统计和语言标签。
- 生成按钮放在右侧顶栏，旁边的文字"生成 3 个版本"让用户知道会发生什么。
- 生成结果：3 列卡片。卡片高度由内容决定（不像 AI 设计中常见的等高卡片）。长文案的卡片更高，短文案的更矮——自然感。
- hover 时操作按钮出现（复制/编辑），编辑态：文字变为可编辑 textarea。
- 复制成功后 toast："已复制"，底部居中，2.5 秒消失。
- 历史记录横向滚动，小卡片预览（只显示前 2 行文字）。

- 完成后视频自动播放，出现下载按钮和保存到资源库按钮。

**设计决策：**
- 中心预览区内容用**白色/浅灰背景**，而不是暗色——因为预览的是电商页面，用户在判断"这个图在 Amazon 上好不好看"。暗色背景下的预览会欺骗视觉判断。
- 预览模板切换是下拉选择，不搞图标网格——效率优先。
- 截图导出：点击按钮直接下载 PNG，不需要预览确认。文件名自动生成（"Amazon预览_20260514_1530.png"）。
- 未选择资源时：预览框显示浅灰虚线占位 + "从左侧选择文案和图片开始预览"。

## 全局 UI 模式

### Loading

| 场景 | 方案 | 说明 |
|------|------|------|
| 登录/保存 | 按钮内 spinner | 14px 旋转环，替换按钮文字 |
| 文案生成 | 骨架卡片 | 和最终卡片同形状的灰色块，pulse 动画 |
| 图片生成 | 骨架图片 | 和最终图片同尺寸的灰色块，pulse 动画 |
| 视频生成 | 进度条 | 2px accent 色细线 + 百分比 + 阶段说明 |
| 页面加载 | 无 | 瞬间显示——好的架构不需要 loading 页面 |

**不用 shimmer。** Shimmer（从左到右的扫光）是 2024 年最泛滥的 AI UI 模式。改用简单的 pulse（不透明度呼吸），更克制。

### 空状态

格式：图标 + "还没有 {内容类型}" + 行动链接

```
例：
  📷
  还没有图片
  去生成第一张 →
```

不用"暂无数据"这种零信息量的文字。告诉用户怎么产生数据。

### 错误

- 网络错误：页面顶部横幅（深红底 + 白色文字），出现"重试"按钮。
- API 错误（生成失败）：卡片内红色文字说明 + "重试"链接。
- 表单验证错误：字段下方红色小字，字段边框变红。
- 不弹 alert()。

### Toast

- 底部居中，距底部 24px。
- 暗底 + 白色文字 + 左侧竖线（成功绿/错误红）。
- 3 秒自动消失，可手动关闭。
- 不叠放——同一时间只显示一个。

### 键盘快捷键

```
⌘/Ctrl + Enter   生成（文案/图片）
⌘/Ctrl + S       保存到资源库
Esc              关闭弹窗/模态窗/lightbox
Space            播放/暂停视频
← →              lightbox 切换图片
```
